# 線上座位表系統規格

## 系統定位

辦公室座位管理系統，以視覺化平面地圖即時反映員工在場狀態。  
管理者負責指派座位，員工透過打卡觸發在場狀態更新。

---

## 使用情境

- 員工上班後可查看同事是否在座位上
- 管理者可統覽全辦公室的出席狀況
- 新人或訪客可快速找到特定員工的位置

---

## 使用者角色

| 角色 | 權限 |
|------|------|
| 員工 | 查看地圖、打上下班卡、查詢其他員工位置 |
| 管理者 / HR | 所有員工權限 + 新增/刪除員工、指派座位 |

- 登入方式：帳號密碼（系統自建帳號，不整合第三方 SSO）

---

## 功能範圍（MVP）

### 1. 座位地圖顯示

#### 1.1 底圖上傳

- 管理者可上傳辦公室平面圖作為地圖底圖，僅支援 **DXF** 格式（AutoCAD 匯出，保留精準向量座標）
- 每個樓層分別上傳一份底圖，樓層之間彼此獨立
- 上傳後系統將 DXF 轉換為可在瀏覽器顯示的向量圖層（如轉為 SVG 或透過 canvas 渲染）
- 底圖僅作為視覺參考，不帶任何業務邏輯

##### DXF 上傳流程

1. 管理者於「樓層管理」頁面點擊「新增樓層」
2. 輸入樓層名稱（例如：`3F 研發部`）
3. 選擇並上傳該樓層的 `.dxf` 檔案
4. 系統解析 DXF 並顯示預覽，管理者確認底圖方向與比例正確
5. 確認後儲存，該樓層底圖正式啟用

##### DXF 解析規則

- 解析 DXF 中的線條（LINE、LWPOLYLINE）與文字（TEXT、MTEXT）圖層作為底圖顯示，底圖固定完整顯示
- 若 DXF 含多個 Layout（模型空間 / 圖紙空間），固定使用模型空間（Model Space）
- 若解析失敗，系統顯示錯誤訊息並擋上傳

#### 1.2 多樓層管理

- 系統支援多樓層，每個樓層為獨立底圖與座位集合
- 管理者可新增、重新命名、刪除樓層
- 刪除樓層前，系統警告該樓層下所有座位與指派關係將一併移除
- 樓層顯示順序可手動拖曳排列

#### 1.3 地圖瀏覽

- 使用者可以用滑鼠滾輪縮放、左鍵拖曳平移瀏覽整張地圖
- 觸控裝置支援雙指縮放與單指拖曳（Web 瀏覽器）
- 提供「重置視角」按鈕，回到預設縮放與置中位置
- 多樓層切換採**輪播方式**：
  - 地圖左上角固定顯示目前樓層名稱
  - 右側提供箭頭按鈕，點擊後切換至下一樓層
  - 樓層順序依管理者設定的排列順序顯示

#### 1.4 座位標記

- 系統在底圖上以圓形或方形圖標標示各個座位位置
- 每個座位標記顯示：
  - 員工頭像（無頭像時顯示姓名首字）
  - 員工姓名（Hover 或點擊後顯示完整資訊）
  - **在場狀態色彩**：
    - 綠色：已打上班卡（在場）
    - 灰色：已打下班卡 / 尚未打卡（不在場）
- 未指派員工的座位顯示為空座位（淺色虛線外框）
- 點擊座位標記後，顯示員工基本資訊浮窗（姓名、部門）

### 2. 打卡功能
- 系統內建上下班打卡，不整合外部考勤系統
- 員工登入後可執行「上班打卡」/ 「下班打卡」
- 打卡後地圖上對應座位標記即時更新顏色

### 3. 座位指派（管理者）
- 管理者可將員工指派到指定座位
- 指派為固定座位（非熱桌制）
- 無需保留座位變更歷史紀錄

### 4. 員工查詢
- 可依姓名搜尋員工，地圖自動定位至該員工座位

---

## 不在範圍內（Out of Scope）

- 熱桌 / 座位預約制
- 座位變更歷史紀錄
- 與外部 HR / 考勤系統整合
- 手機 App（僅支援 Web 瀏覽器）
- 多公司 / 多地點（Multi-site）支援
- 報表與統計功能

---

## 地圖設定流程（管理者）

1. 新增樓層，輸入樓層名稱
2. 上傳該樓層的 `.dxf` 檔案作為底圖，確認預覽正確
3. 在底圖上點擊新增座位標記，輸入座位編號
4. 將員工指派到對應座位
5. 重複步驟 1–4 完成所有樓層設定

---

## 狀態流程

```
員工登入
   ↓
執行「上班打卡」
   ↓
地圖座位標記 → 綠色（在場）
   ↓
執行「下班打卡」
   ↓
地圖座位標記 → 灰色（不在場）
```
